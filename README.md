# F1Telemetry

## 项目介绍

F1Telemetry 是一个 **F1 25 Windows 遥测助手**。它接收游戏 UDP 遥测数据，把实时比赛状态整理成中文概览、AI 分析播报、语音输入 AI 问答、单圈历史、对手信息、事件日志，并可通过 DeepSeek AI 和 Windows TTS 给出短提示。

当前版本：`3.0.4`。项目已具备 V3 基础能力，包括 AI/TTS 播报节流、语音输入 AI 问答、赛后 AI 详细报告、ERS 格式化、结构化轮胎库存、胎磨图表配方着色、弯角级分析和策略复盘基础能力。

## 项目现有功能

- UDP 接收：监听 F1 25 遥测端口并解析主要比赛数据。
- 实时概览：显示赛道、赛制、轮胎、燃油、ERS、前后车差距和关键事件摘要。
- AI 分析播报：将当前圈速度、油门/刹车、多圈燃油和四轮磨损趋势压缩进 AI 短结论，并通过 TTS 播报关键提醒。
- 语音输入 AI 问答：支持绑定方向盘/手柄按钮，按住或切换录音后识别车手问题，结合当前比赛上下文调用 AI，并通过 TTS 播报短答。
- 单圈历史：记录当前实时会话最近圈速、有效圈、燃油消耗和轮胎磨损摘要。
- 历史会话浏览：异步加载 SQLite 中的历史 session，并用分页查看/删除历史 session 与选中 session 的单圈列表。
- 赛后复盘：从历史 laps / events / ai_reports / lap_samples 加载会话摘要、深色趋势图、按轮胎配方着色的四轮胎磨趋势、事件时间线和 AI 每圈点评。
- V3 弯角分析：基于已持久化 lap samples 和赛道分段估算弯角速度、损失、置信度与数据质量限制，可选择历史会话和圈号。
- V3 策略时间线：从历史圈、轮胎阶段和关键事件生成 stint / 安全车 / 红旗影响时间线，作为赛后复盘补充。
- V3 工程师报告基础：使用压缩摘要生成可保存的赛后工程师报告输入，不发送原始高频 UDP、API Key 或 Header。
- 多会话对比：按同赛道筛选并分页选择 2-4 个历史 session，对比圈速、燃油、ERS 和按轮胎配方着色的四轮平均胎磨趋势。
- 复盘报告导出：从赛后复盘页导出 Markdown / JSON 报告，不包含 API Key 或原始高频 UDP。
- 对手信息：展示可读化的对手状态、进站、遥测限制和差距。
- 事件日志：按 System / UDP / RaceEvent / AI / TTS / Storage 分类查看完整日志。
- AI 分析：使用 DeepSeek OpenAI compatible endpoint 生成短结论，并结合天气、进站窗口、排名、前后车、胎磨和赛前轮胎库存约束策略建议。
- Windows TTS：使用本机 Windows 语音播报关键事件和 AI 结论。
- SQLite 持久化：保存会话、单圈、事件、AI 报告和 TTS 记录。
